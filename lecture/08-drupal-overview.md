# ステップ 8: Drupal がページを返すまでの全体像

ここまでの知識を組み合わせると、Drupal がブラウザにページを返すまでの全体像が見えてくる。

## リクエストからレスポンスまでの流れ

```
[ブラウザ]
  ↓  リクエスト（例: GET /node/1）
[Web サーバー (Apache/Nginx)]
  ↓  すべてのリクエストを index.php に転送
[index.php]  ← Drupal のエントリポイント（1 つだけ）
  ↓
[カーネル起動]  ← Drupal のコア部分を起動
  ↓
[ルーティング]  ← URL を解析して、どのコントローラを呼ぶか決定
  ↓
[コントローラ]  ← リクエストに応じた処理（データ取得など）
  ↓
[レンダリング]  ← テーマ → テンプレート（Twig）→ PHP 実行 → HTML 生成
  ↓
[レスポンス]   ← 生成された HTML をブラウザに返す
  ↓
[ブラウザ]     ← HTML を受け取って画面に描画
```

## 静的 HTML との比較

| | 静的 HTML | Drupal |
|---|---|---|
| サーバー上のファイル | ページごとに `.html` ファイルがある | `.php` と `.html.twig` ファイルがある |
| URL とファイルの関係 | URL = ファイルパス | URL とファイルパスは無関係 |
| ページの内容 | ファイルの中身そのまま | データベースの内容 + テンプレート |
| ページ追加 | HTML ファイルを新規作成 | 管理画面からコンテンツを作成 |
| デザイン変更 | 各 HTML ファイルを個別に編集 | テンプレートを変更すれば全ページに反映 |

## 重要なポイント: 最終的にブラウザに届くのは HTML

Drupal がどれだけ複雑な処理をしていても、最終的にブラウザに届くのは普通の HTML（+ CSS + JavaScript）。ブラウザの「ページのソースを表示」で見ると、静的 HTML で書いたものと変わらない HTML が返ってきている。

```
静的 HTML:   HTML ファイル → そのまま返す
Drupal:      PHP + Twig + DB → 処理して HTML を生成 → 生成結果を返す
```

どちらの場合もブラウザが受け取るのは HTML。違いは「その HTML がどうやって用意されたか」だけ。
